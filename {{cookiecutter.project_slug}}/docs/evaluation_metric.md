# Evaluation Metric

> **作成方法**: 以下のプロンプトをAIに入力し、出力結果を「---」以降に貼り付けてください。
> 入力として、コンペの評価指標名（例: "QWK", "RMSE", "MAP@3" など）を渡してください。
> `docs/competition_overview.md` に記載した情報があれば、併せて渡すとより正確な出力が得られます。

```
あなたは経験豊富なKagglerです。
以下の評価指標について、Kaggleコンペで実際に使う観点から日本語で詳しく解説してください。

# 出力フォーマット

## 指標の定義
- 正式名称
- 数式（LaTeX 形式で記載）
- 直感的な説明（何を測っているのか、高い/低いほうが良いのか）

## スコアを改善するためのアプローチ
- この指標に効果的な一般的手法（閾値最適化、アンサンブル、後処理など）
- 損失関数との関係（学習時にどの損失関数を使うべきか）

## 注意点・よくある落とし穴
- この指標特有の罠（クラス不均衡への感度、外れ値への感度など）
- やりがちなミス

## 実装例
- scikit-learn や numpy を使った計算コード（Python）

# 入力
評価指標: {ここに評価指標名を記載}
コンペの補足情報（あれば）: {competition_overview.md の内容など}
```
